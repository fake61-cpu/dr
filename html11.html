Amazon EC2

✅ Theory: What is Amazon EC2?

Amazon EC2 (Elastic Compute Cloud) is a cloud service that provides virtual servers, known as EC2 instances, to run applications without needing physical hardware. It allows users to choose the operating system, CPU, memory, storage, and network settings as per their requirement.

Key Features of EC2

  • Scalable → Increase or decrease instance capacity anytime.
  • Pay-as-you-go → You pay only for the time the instance is running.
  • Secure → Controlled using key pairs and security groups (firewall rules).
  • Customizable → Choose OS (Linux/Windows), instance type, storage, etc.

✅ Steps to Launch and Configure a Virtual Machine using Amazon EC2

1. Log in to AWS Management Console

Go to: https://console.aws.amazon.com
Search for EC2 in the services menu and open the EC2 Dashboard.

2. Click on “Launch Instance”

This starts the virtual machine (instance) creation process.

3. Choose an Amazon Machine Image (AMI)

An AMI is a preconfigured operating system template.

Common choices:

  • Amazon Linux 2023
  • Ubuntu Server
  • Windows Server

Select the AMI you want to run.

4. Choose Instance Type

This defines your VM’s hardware configuration.

Examples:

  • t2.micro / t3.micro → Free-tier eligible (1 vCPU, 1 GB RAM)
  • Larger instance types are used for heavy workloads.

Select t2.micro for free-tier.

5. Configure Key Pair (Login Method)

A key pair is used to securely connect to your EC2 instance.

  • Click Create New Key Pair
  • Download the .pem file

This file is required for SSH login.

6. Configure Security Group

A security group works like a virtual firewall for your instance.

Add inbound rules:

  • SSH (port 22) → For Linux login
  • HTTP (port 80) → If hosting a website

Keep outbound rules as Allow All.

7. Configure Storage

Choose disk size. The default 8 GB EBS volume is enough for free-tier usage.

8. Review and Launch

Review all settings (AMI, instance type, key pair, security group, storage), then click Launch Instance.
AWS will create your virtual machine in a few seconds.

9. Verify Your Instance

Go to Instances → Running Instances
Check that the instance status shows:

  • Running
  • Status Checks Passed

This confirms that the VM is ready to use.

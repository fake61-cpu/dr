Theory

A Service Level Agreement (SLA) in cloud computing defines performance expectations such as uptime and response time between a provider and user. SLA monitoring helps ensure services meet these standards by measuring performance and detecting failures or slow responses.

In this practical, a Python script is used to monitor service response times using the requests module. Each endpoint is tested, and the response time is compared with a predefined threshold. If the response time is within the limit, the service meets the SLA; otherwise, it is marked as a breach.

Short Code Explanation

  • A list of service URLs is defined.
  • A response time limit (threshold) is set.
  • The script sends requests to each URL and measures time taken using time.time().
  • It prints response status and checks whether it meets or violates the SLA.
  • Errors are handled using a try-except block.

Conclusion

This experiment demonstrated how to monitor SLA compliance by checking the response time of different endpoints. The script successfully identified which services met the performance threshold and which did not, showing how SLA monitoring helps maintain reliable and high-performing cloud services.

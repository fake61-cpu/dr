Creating an Amazon S3 Bucket

Theory

Amazon S3 (Simple Storage Service) is an object-storage service provided by AWS. It allows users to store and manage data such as images, videos, documents, backups, and static website files in a scalable and highly durable manner. S3 stores data inside Buckets, which act as containers for objects. Each bucket has a unique name across AWS and can be configured with permissions, versioning, encryption, and access control.

Key Features of Amazon S3

  • Scalability: Automatically handles any amount of data.
  • Durability: Redundant storage across multiple Availability Zones (99.999999999% durability).
  • Accessibility: Objects can be accessed via URL, API, or SDK.
  • Security: Supports IAM roles, bucket policies, ACLs, and encryption.
  • Use Cases: Website hosting, backups, data lakes, media storage, and app content.

Steps to Create an S3 Bucket

1. Open AWS Management Console

Log in to your AWS account. From the services menu, search for S3 and click on Amazon S3.

2. Create a New Bucket

Click Create bucket.

3. Enter Bucket Name and Region

Provide a unique bucket name (example: my-first-s3-bucket-lab123). Select an AWS region near you for best performance.

4. Configure Bucket Options

You may choose:

  • Versioning → Keeps multiple versions of objects.
  • Encryption → Enable SSE-S3 or SSE-KMS for security.
  • Object Lock (optional) → Prevent deletion of objects.

For basic labs, the default settings are fine.

5. Set Permissions

Uncheck Block all public access only if the bucket needs to be public (for example, website hosting). Otherwise keep the bucket private (recommended).

6. Create the Bucket

Scroll down and click Create bucket. The new bucket will now appear in your list.

7. Upload Files (Optional)

Open your bucket → click Upload → Add files → and upload any file.

8. (Optional) Enable Static Website Hosting

Go to Properties → Static website hosting → Enable. Upload index.html and update permissions accordingly.

Conclusion

In this lab, we successfully created an S3 bucket using the AWS Management Console. We learned how to configure region, permissions, security settings, and upload files. This helps us understand cloud storage operations and how Amazon S3 is used for hosting, data storage, and application support.

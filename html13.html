Launch a Website Using an EC2 Instance

Theory

Amazon EC2 (Elastic Compute Cloud) provides virtual machines called instances. These work like remote computers where applications, software, and websites can be hosted without physical hardware. Users can choose an operating system, configure CPU/RAM, attach storage, and manage network settings.

To host a website, we launch a Linux instance, install a web server such as Apache or Nginx, and upload website files. To make it publicly accessible, HTTP traffic (Port 80) and SSH access (Port 22) must be allowed in the Security Group.

‚≠ê Steps to Launch a Website Using EC2

Step 1 ‚Äî Log in to AWS Console

Visit: https://aws.amazon.com/console
Search for EC2 and open the EC2 Dashboard.

Step 2 ‚Äî Launch a New EC2 Instance

  • Name: MyWebServer
  • AMI: Amazon Linux 2 / Ubuntu (Free-tier eligible)
  • Instance Type: t2.micro

Step 3 ‚Äî Configure Key Pair

Create a new key pair ‚Üí Download the .pem file. (Required for SSH login)

Step 4 ‚Äî Configure Network Settings (Security Group)

Add inbound rules:

Type Port Description
SSH 22 For remote login
HTTP 80 Website access
HTTPS (Optional) 443 Secure access

Keep outbound rules default (allow all).

Step 5 ‚Äî Launch the Instance

Click Launch Instance and wait for status to turn Running.
Copy the Public IPv4 address or Public DNS.

Step 6 ‚Äî Connect to the Instance (SSH)

Mac/Linux:

ssh -i file.pem ec2-user@your-public-ip

Windows: Use PuTTY (convert .pem ‚Üí .ppk using PuTTYgen).

Step 7 ‚Äî Install a Web Server

For Amazon Linux:

sudo yum update -y
sudo yum install httpd -y
sudo systemctl start httpd
sudo systemctl enable httpd

For Ubuntu:

sudo apt update
sudo apt install apache2 -y
sudo systemctl start apache2
sudo systemctl enable apache2

Step 8 ‚Äî Add Your Website Files

Edit webpage:

sudo nano /var/www/html/index.html

Example content:

<h1>Hello from my EC2 Website!</h1>

Step 9 ‚Äî View the Website

Open:

http://<your-public-ip>

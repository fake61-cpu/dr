Theory

In cloud computing, virtualization is the process of creating virtual versions of computing resources like servers, storage, or networks, allowing multiple Virtual Machines (VMs) to run on a single physical system for better resource utilization, flexibility, and scalability. Each VM operates independently with its own OS and applications, managed by a hypervisor that allocates hardware resources such as CPU, memory, and storage.

There are two types of hypervisors:

  • Type-1 (Bare Metal): Runs directly on hardware (e.g., VMware ESXi, KVM)
  • Type-2 (Hosted): Runs on a host operating system (e.g., VirtualBox, VMware Workstation)

libvirt, an open-source API, allows managing VMs through Python bindings for tasks like creating, starting, and monitoring VMs—essential for cloud automation. When real VM creation isn’t possible, VM scheduling simulation can be used to study resource allocation using algorithms such as FCFS, Round Robin, Best Fit, and Load Balancing, helping analyze efficiency and performance in virtualized cloud environments.

Short Code Explanation

  • VMRequest stores VM requirements like CPU, memory, and duration.

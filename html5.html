Theory

Load balancing distributes network traffic across multiple servers to prevent overload and improve performance, reliability, and resource utilization. Two common algorithms used are:

  • Round-Robin: Assigns requests to servers in a cyclic order; simple and effective when servers have equal capacity.
  • Least-Connection: Sends new requests to the server with the fewest active connections; ideal for uneven workloads and ensures balanced utilization.

Short Code Explanation

  • ServiceInstance represents a backend server handling requests.
  • RoundRobinLoadBalancer stores service instances and assigns requests in cyclic order.
